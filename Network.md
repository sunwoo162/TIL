# Network
### IO : 입출력 프로그래밍
- 스트림이란?
	- 데이터의 흐름
	- 입출력 스트림은 입출력 장치와 프로그램 사이의 일련의 데이터 흐름을 의미
	- 스트림을 통해 흘러가는 데이터의 기본 단위는 바이트
- 스트림 : 입출력한 자료가 흘러가는 통로
	- 스트림은 객체화 -> 클래스로 만들어져 있다.
	- In~~~//Read~~~ : 입력
	- Out~~~// Write ~~~ : 출력
	- 단반향성
### 입출력 데이터의 종류
1. 바이트 스트림(1Byte) : 영문자, 이미지, 동영상, 음악 등 파일 전송
- 입력 : InputStream ----> Read()
- 출력 : OutputStream ----> Write()
2. 문자 스트림 (2byte) : 텍스트(한글, 특수문자, 한자 등)
----> UniCode(UTF-8, MS949)
- 입력 : InputStream ----> Read()
- 출력 : OutputStream ----> Write()

### 문자열
BCD : 6bit -> 64문자
ASCII : 7bit -> 128문자 + 1bit(Party bit : 오류검출코드)
EBCDIC : 8bit -> 256문자
우리는 왜? 아스키코드를 사용할까
중간에 에러 방지 1bit짜리 문자를 넣어둠 그래서 아스키코드 사용

### 스트림 종류
1. 노드 스트림 : 가장 먼저 연결되는 스트림
2. 필터 스트림 : 노드 스트림과 연결되는 스트림 -> 브릿지 스트림(연결자)
	- 브릿지 스트림은 단독으로 사용할 수 없다 : 노드 스트림과 연결
	- InputStreamReader : 바이트 입력 스트림을 문자로 변형
	- OutputSreamWriter : 바이트 출력 스트림을 문자로 변형
